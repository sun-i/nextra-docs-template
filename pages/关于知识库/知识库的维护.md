# Github pages
[https://pages.github.com/](https://pages.github.com/)
> 你可以为我讲讲github page吗  

GitHub Pages 是一个功能，允许用户利用 GitHub 存储库中的内容来托管个人、项目或组织的网站。这是一个免费服务，对于展示项目、托管博客、简历或其他个人项目的静态网页非常有用。下面是有关 GitHub Pages 的一些关键点：

1.  **静态网站托管**：GitHub Pages 专门用于托管静态网站，这意味着你可以托管 HTML、CSS 和 JavaScript 文件，但不能执行服务器端脚本，如 PHP 或 Python。 
2.  **使用 Jekyll**：GitHub Pages 内置支持 Jekyll，这是一个流行的静态网站生成器。你可以使用 Jekyll 来自动生成网站布局和样式，而无需手动编写大量的 HTML 代码。 
3.  **个性化域名**：虽然每个 GitHub Pages 站点都会提供一个 `username.github.io` 的免费域名，但用户也可以配置自己的自定义域名来个性化自己的网站。 
4.  **版本控制**：由于 GitHub Pages 是直接与 GitHub 存储库集成的，每一个网站的更改都会通过 git 进行版本控制，这使得回滚更改和版本管理变得非常简单。 
5.  **自动部署**：每当你推送新的代码到特定的 GitHub 存储库（通常是 `master` 分支或 `gh-pages` 分支）时，GitHub Pages 会自动重新部署你的网站，这意味着更新你的网站内容非常方便。 
6.  **安全性**：GitHub Pages 提供 HTTPS，确保你的网站内容在传输过程中的安全性。 

设置 GitHub Pages 非常简单，只需将你的静态文件推送到指定的 GitHub 存储库分支，然后在仓库的设置中启用 GitHub Pages 功能即可。对于需要简单、易于维护且几乎不涉及成本的个人或小型项目网站来说，这是一个极好的选择。
> 这种托管需要付费吗

GitHub Pages 的基本使用是完全免费的。这包括托管静态网站，无论是个人的、项目的还是组织的网站。这意味着你可以托管一个网站，使用 GitHub 提供的 `username.github.io` 域名，不需要支付任何费用。

然而，如果你需要额外的功能，如更多的存储空间或更多的数据传输，你可能需要升级到付费的 GitHub 计划。例如，免费计划提供的带宽和存储空间对于大多数小型和个人项目来说是足够的，但对于流量较大或需要存储大量内容的网站，可能需要更高级的计划。

此外，如果你想为你的 GitHub Pages 网站使用自定义域名，你需要自己购买和维护域名，这通常需要向域名注册商支付年费。

总的来说，对于基本用途，GitHub Pages 是一个免费的解决方案，非常适合个人、开源项目和教育用途。对于更复杂或资源需求更高的使用情况，可能需要考虑其它的付费选项。

# 关于运营成本
一些可能的选择

1. 众筹：视情况设定一个众筹额度，筹够就开项目，筹不够就搁置
2. 捐赠：看大家愿不愿意提供捐赠，但捐赠就会涉及到谁来管理账目的问题
3. 站长个人负担：站长个人负担服务器开销，但捐赠也归站长所有，盈亏都由站长承担，但如果盈利的话，也有很多人贡献了内容，如何分配收入给贡献者呢，这可能会导致大家心态失衡
